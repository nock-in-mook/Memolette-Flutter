# Memolette Flutter版 — プロジェクト指示書

## プロジェクト概要
Swift版メモアプリ「Memolette（即メモ君）」をFlutterで再構築するプロジェクト。
Swift版のソースコード・設計ドキュメントを参照しながら移植する。

## 技術選定（確定済み・変更不可）
- **データ保存**: Drift（SQLite）— 多対多リレーション対応、同期の自由度が高い
- **状態管理**: Riverpod — 型安全、テストしやすい
- **同期方針**: SQLiteをローカルの「信頼できる唯一のデータ源」にし、Firebase/iCloudはそこに同期
  - Firebase同期: Firestore ↔ SQLite
  - iCloud同期: CloudKit ↔ SQLite
- **画像挿入**: 圧縮+リサイズ方式（長辺1024px, JPEG 70%, 1枚100-200KB）

## 移植順（予定）
1. Memo CRUD + ローカル保存（Drift）
2. Tag管理（親子階層、多対多）
3. メモ一覧画面（グリッド表示）
4. メモ入力・編集画面
5. 爆速モード
6. マークダウン対応
7. ToDo機能
8. Firebase同期
9. iCloud同期

## UIルール（厳守）
- **ダイアログは全てカスタムリッチUI** — 標準の `showDialog` / `AlertDialog` は使わない
- **設計方針**: UIは常にシンプルを保つ。機能が増えてもカオスにしない
- **コンセプト**: 普通の人にもものすごく使いやすく、パワーユーザーの期待も大きく上回る
- **タブ内のメモ一覧は「バッグ」と呼ぶ**（開発用語として統一）

## 設計上の注意
- **画面端スワイプ**: iOS左端=戻る、Android両端=ナビゲーション。端からの引き出し操作はOS干渉するため、タブタップをメイン操作にする
- **絶対値レイアウト禁止**: 画面サイズ依存のハードコードを避け、比率ベースで実装する（Swift版ではこれが課題だった）
- **自動保存**: ユーザーが入力するたびに即座に保存（debounceなし）
- **ソート基準**: createdAt降順。updatedAtは編集時のみ更新し、閲覧ではソート順を変えない

## 多言語対応方針（確定）
- **フェーズ1リリース時**: 日本語、英語、中国語（簡体）、スペイン語、フランス語
- 文字列はキー参照（直書き禁止）。FlutterではARBファイル使用
- アイコン＋短いラベルを1パターンで統一
- アイコンのみボタンにはセマンティクスラベル必須

## 収益モデル
- 無料版: 基本機能（一部上限あり）
- Pro（サブスク）: 上限解放＋AI機能＋クラウド同期
- 広告なし

## Swift版で発生したバグ一覧（Flutter版で同じ轍を踏まないこと）

全59セッションから抽出した30件。**特にジェスチャー競合（6件）、キーボード関連（4件）が頻出**。

### 3大注意ポイント
1. **ジェスチャー競合の管理** — ドラッグ・タップ・スワイプが共存するUIではGestureDetectorの優先度とHitTestBehaviorを慎重に設計
2. **キーボードとダイアログの共存** — ダイアログ表示前にFocusScope.of(context).unfocus()でキーボードを閉じるのを標準化
3. **状態変更の副作用管理** — onChangeやListenerのチェーンで意図しない画面遷移/データ更新が起きないよう、フラグやガードを設ける

### バグ詳細

#### #1: タグ保存バグ（セッション003）
- タグ保存時に正しく保存されない
- **教訓**: タグの多対多リレーション保存ロジックはテスト必須

#### #2: ビルドキャッシュ問題（セッション005, 017）
- コード変更がビルドに反映されない
- **教訓**: `flutter clean` を習慣化。特にネイティブプラグイン変更時

#### #3: ボタンテキストのデフォルト色問題（セッション007）
- Button内テキストが意図せずデフォルト色になる
- **教訓**: カスタムボタンでは明示的にスタイルを指定する

#### #4: 子タグがフォルダタブに表示される（セッション009）
- 親タグ専用タブに子タグも混入
- **教訓**: 親子タグのフィルタリングは表示箇所ごとに一貫してフィルタ適用

#### #5: 「タグなし」選択時のタブ遷移バグ（セッション012）
- タブインデックスがsortOrderベースでなく同期ずれ
- **教訓**: 特殊タブ（「すべて」「タグなし」等）のインデックス管理は明示的に分離

#### #6: ルーレット「+追加」マスの誤発火（セッション015）
- ルーレット回転中に「+追加」に止まると新規タグシートが出る
- **教訓**: スクロール可能なUI上にアクション発火要素を置かない。操作エリアとアクションボタンは分離

#### #7: ツールバーエリアのスワイプでフォルダ移動（セッション017）
- ジェスチャー範囲がツールバーにも及んでいた
- **教訓**: GestureDetectorのスコープを明確に限定する

#### #8: ボタンのタップ領域と見た目の不一致（セッション017）
- タップしても反応しない領域がある
- **教訓**: ボタンのhitTest範囲は見た目と一致させる。InkWellのcustomBorderやパディングに注意

#### #9: clearInput後にタグなしフォルダへ勝手に移動（セッション017）
- clearInput時のonChangeがフォルダ自動移動をトリガー
- **教訓**: onChangeの副作用を最小限に。自動移動は原則避け、ユーザーの明示的操作のみ

#### #10: メモソートが閲覧だけで変わる（セッション017）
- updatedAtベースソートで閲覧=更新扱いされた
- **教訓**: ソートはcreatedAt。updatedAtは編集操作時のみ更新

#### #11: 「ここに保存」誤タップ（セッション018）
- ボタン配置と操作動線が近すぎた
- **教訓**: 保存・削除等の重要操作には確認ダイアログを設ける

#### #12: 子タグ連打フリーズ（セッション018）
- 連続タップによる再描画の競合
- **教訓**: タップイベントのデバウンス処理。アニメーション中のタップを無視するガード

#### #13: シート表示時の伸び縮み（セッション030, 031）
- アニメーション競合（シミュレータ固有の場合も）
- **教訓**: シミュレータと実機の両方で確認。BottomSheetのアニメーション競合に注意

#### #14: フォルダタブ上部のクリップ切れ（セッション036）
- frame高さ・padding不足
- **教訓**: タブUIのclipBehaviorを意識。Clip.noneが必要な場面を把握

#### #15: UTF-16インデックスバグ（セッション035）
- 絵文字等でタグ名比較のインデックスがずれる
- **教訓**: Dartでは`characters`パッケージでグラフィムクラスタ単位で扱う

#### #16: ルーレットスナップ不具合（セッション041）
- ドラッグとタップのジェスチャーが同時発火
- **教訓**: ドラッグ終了後のクールダウン期間（0.5秒ガード）を設ける。GestureDetectorの優先度管理

#### #17: ボタンのトグル二重発火（セッション044）
- タップイベントの重複
- **教訓**: カスタムボタンのジェスチャーハンドラは1つに統合

#### #18: サードパーティキーボードでカーソル行が隠れる（セッション045）
- キーボード高さの伝播が途切れた
- **教訓**: `MediaQuery.of(context).viewInsets.bottom` で統一。ネストウィジェットでも正しく伝播されるか確認

#### #19: 閲覧/編集モードのテキスト位置ズレ（セッション046）
- UITextViewのインセットが閲覧/編集で異なっていた
- **教訓**: 表示用と編集用のパディング・フォント・行間を完全一致させる

#### #20: キーボードとダイアログの押し潰れ（セッション048）
- キーボード+ダイアログ同時表示でレイアウト破綻
- **教訓**: ダイアログ表示前に `FocusScope.of(context).unfocus()` を標準化

#### #21: ToDo追加行の文字列重複（セッション048）
- onSubmitとonChangedの競合
- **教訓**: テキスト入力のonSubmitとonChangedの競合に注意。連続入力UIでは完了タイミング管理が重要

#### #22: チェックボックスタップ判定の不具合（セッション050）
- タップ判定領域の設定不備
- **教訓**: チェックボックスは最低44x44ptのタップ領域を確保

#### #23: メモ追加時のonChange競合でタグが反映されない（セッション028）
- 複数Listenerの更新順序競合
- **教訓**: 複数Listenerが同一状態を監視する場合、更新順序と競合を意識

#### #24: 入力欄最大化ボタンの不具合（セッション028）
- **教訓**: 拡大/縮小トグルは単一bool値で管理、アニメーション中のタップを無視

#### #25: DragGesture + ScrollView = スクロール不能（セッション049）
- DragGestureがScrollViewのジェスチャーを奪う
- **教訓**: FlutterのReorderableListViewを使う。カスタムドラッグはスクロール競合に注意

#### #26: 爆速モード削除後の位置維持バグ（セッション053）
- カルーセルのインデックス管理の問題
- **教訓**: リスト/カルーセルからアイテム削除後のインデックスを再計算する

#### #27: 爆速モード結果画面のチラ見え（セッション053）
- 画面遷移のタイミング制御不足
- **教訓**: Navigator.pushの遷移タイミングを制御し、不要フレームが見えないようにする

#### #28: 枠外タップでキーボード閉じるとボタンタップ阻害（セッション058, 059）
- キーボード閉じジェスチャーがボタンタップを横取り
- **教訓**: GestureDetectorの `behavior` を `HitTestBehavior.translucent` にして子ウィジェットのタップを阻害しない

#### #29: テキスト入力時のグレーアウト（セッション058）
- 入力中テキスト属性の設定不備
- **教訓**: TextFieldのスタイルが動的に変わる場面では入力中のスタイルも明示的に設定

#### #30: タイトル編集中の確定ボタン不具合（セッション059）
- **教訓**: フォーカス管理と確定ボタンの動作を統一。タイトル/本文どちらにフォーカスがあっても同じ動作
