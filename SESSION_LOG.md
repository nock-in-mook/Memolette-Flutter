## 1. 基本要件（必須）

| 領域 | ポイント |
|------|---------|
| **レスポンシブ** | `MediaQuery.shortestSide >= 600` でタブレット判定。LayoutBuilder でブレークポイント設計 |
| **横画面スプリット** | 左：タブ＋一覧（バッグ）／右：メモ詳細。`NavigationSplitView` 相当の構成 |
| **縦画面** | iPhone同様の単列 or サイドバードロワー（選択要） |
| **Info.plist** | `UIRequiresFullScreen=NO`、`UISupportedInterfaceOrientations` を iPad 向けに全方向対応 |
| **絶対値→相対値** | ROADMAPにも残ってる課題。iPadでは間延び or 破綻が出やすい |

## 2. 盲点になりやすい所

- **Multitasking 対応**: Split View / Slide Over / Stage Manager で**画面幅が動的に変わる**。起動中にリサイズされても破綻しないか要検証（これが最大の盲点）
- **フローティングキーボード**: iPhoneと違って画面内任意位置。`MediaQuery.viewInsets.bottom` が0でも隠れることがある
- **BTキーボード / トラックパッド / Apple Pencil**: MouseRegion hover、secondary tap（右クリック相当）、Pencil Scribble対応
- **ルーレットUI**: 指で扇形回すUI、iPadだと腕動作が大きくなる → 片手の親指で届く位置に寄せる工夫が必要
- **タブの無限スクロール**: iPad幅だとタブが余る → 見せ方（全表示 vs スクロール）を再設計
- **Safe Area / Home indicator**: 縦横でダイナミックに変わる。ルーレットやフッターの位置に影響
- **画像サムネの密度**: 画面が広い分、小サムネが点のように見える恐れ

## 3. 入れたら充実するやつ

### 🎯 高コスパ（実装軽め・インパクト大）

- **ドラッグ&ドロップ**: 他アプリから画像/テキストを投げ込んで即メモ化
- **右クリック / 長押しでコンテキストメニュー**: 編集・削除・タグ付けを一発

### 🌟 ROADMAPと合流できる
- **並列編集モード**（ROADMAPアイデアメモ）: 画面を左右分割→別メモを並べて見る。iPadならではの強力UX
- **サイドバー常時表示**: 親タグ一覧を左固定。iPad標準の「NavigationRail」的な佇まい

### 💎 Proっぽさを出せる

| **Docked**（通常） | キーボード高さ | iPhoneと同じ、そのまま動く ✅ |
| **Split**（左右分割の小型） | 小さい値 | 同上、問題なし ✅ |
| **Floating**（iPhoneサイズで浮いてる） | **0** | **完了バーが画面下端に取り残される** ⚠️ |
| **外部BTキーボード** | **0** | 完了バーを出す意味なし（押下するマウス操作が必要） ⚠️ |
| **Magic Keyboard（ケース一体）** | **0** | 同上 ⚠️ |

## 現状の仕組み

このアプリは `KeyboardDoneBar` でキーボード直上にバーを出している（memory の「Keyboard Dismiss」ルール）。iPhoneでは「キーボードが出ていれば viewInsets.bottom > 0」なので、それに合わせてバーを配置。

**iPad で出る3つの挙動**：
- Docked/Split → **問題なく動作**
- Floating → ソフトキーボードがあるのに viewInsets.bottom = 0 → 完了バーが画面最下部に固定されて不自然
- 外部キーボード → そもそも完了バー不要、UI が邪魔

## 対応案（3段階）

### 最低限（今セッションでやる価値あり）
- `viewInsets.bottom == 0 && isTextFieldFocused` のとき = 外部キーボード or Floating 判定
- **完了バーとキーボード上ツールバーを非表示**にする
- 代わりに画面下部（Safe Area内）に「閉じる」ボタンや画面タップで閉じる代替

### 中（Step C で対応）
- **⌘キーショートカット**で完了・MD操作を代替（外部キーボード前提ユーザー向け）
  - `⌘return` で完了、`⌘B` で太字、`esc` でフォーカスクリアなど
- これで外部キーボード使用時は完了バーなしでも不便なし

### 応用（余力があれば）
- Floating キーボード位置を取得する方法はないが、**ユーザーが自分で動かせる前提**なので、完了バーを画面下固定でOKにしてしまう割り切りもアリ

---
