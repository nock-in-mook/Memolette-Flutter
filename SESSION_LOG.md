- **Alt** → Option
- **Ctrl** → Control

- 印字が「Win」のまま見た目には違和感

- US 配列の Windows キーボードなら印字と動作のズレが少なめ

iPad の設定 → 一般 → キーボード → ハードウェアキーボードで配列・修飾キーのカスタマイズも可能（例: Caps を ⌘ にするとか）。

## 「1: 統合」の実装解釈

ToDoアイテムは既に `dueDate` を持っているので、これをカレンダー紐付け日として **流用** するのが自然。Memos / TodoLists には `eventDate` を新設。「期限」という概念は廃止して、UI ラベルは全部「日付」に統一。

懸念点:
- 既存ユーザーが `dueDate` を「期限」として使ってた場合、ラベルが「日付」に変わる（実害は少ないが)
- カラム名が DB 上は `dueDate` (TodoItems) と `eventDate` (Memos/TodoLists) で混在 → コードはちょっとうるさいけど data migration リスクなし

別案: TodoItems の `dueDate` も `eventDate` にリネーム（migration で既存データを移行）。スキーマ綺麗、ただ data migration 工数+リスクあり。

**A. dueDate そのまま流用 (シンプル、既存データ無傷)** 
**B. 全部 eventDate にリネーム (スキーマ綺麗、migration 慎重)**
