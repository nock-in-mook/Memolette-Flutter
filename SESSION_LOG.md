これは Flutter のフォーカス復元仕様の副作用で、**モーダル / 画面が閉じると pop 直前のフォーカス（メモ入力欄）が自動復元される** のが原因です。`focusSafe` でラップしてる箇所だけは抑えてるけど、抑え漏れがあると毎回起きる。

**根本対策**: `MaterialApp` に **`NavigatorObserver` を 1 つ仕込んで、didPop 時に強制 `unfocus()`** を呼ぶ。これで全画面の遷移後に一律フォーカスが外れる。一カ所追加で全部解決。

```dart
class _UnfocusOnPopObserver extends NavigatorObserver {
  @override
  void didPop(Route route, Route? previousRoute) {
    super.didPop(route, previousRoute);
    WidgetsBinding.instance.addPostFrameCallback((_) {
      FocusManager.instance.primaryFocus?.unfocus();
    });
  }
}
```

**副作用（許容範囲だと思う）**:
- 編集中にダイアログ／シートを出して閉じると、編集続行のためのフォーカスが外れる
- → ユーザーがもう一度 TextField タップすれば戻る。「閉じたらキーボード閉じる」直感的挙動

副作用が困る箇所（編集中の確認ダイアログとか）は、HANDOFF に書いてある「`focusSafe` を使わない + `_isDialogOpen` フラグ」設計のままで対応できます（pop しないで Navigator.of(context).pop() ではなく setState で閉じる、みたいなパターン）。
