## 設計方針（Swift版から引き継ぎ・絶対的命題）
- **UIは常にシンプルを保つ** — 機能が増えてもカオスにしない
- 普通の人にもものすごく使いやすく、パワーユーザーの期待も大きく上回る
- 「鳥肌が立つほど便利！」と言わせるアプリにする
- タブ内のメモ一覧は「バッグ」と呼ぶ（開発用語として統一）

## Phase 9: 同期

### ★★★ データ保護方針（同期実装の必須要件・絶対遵守）★★★
他アプリで「ローカル→クラウド移行で全データ消失」の事故事例あり。
**バグは許容できるがデータ消失だけは死んでも起こさない**。同期機能は以下を全て満たさない限りリリース不可。

#### 設計原則
- **同期は「複製」、移行ではない**：ローカル DB は **絶対に消さない**。クラウドは常にコピー先
- **片側破損で全滅しない**：ローカルが壊れても、クラウドが壊れても、もう一方から復旧できる前提に
- **デフォルト挙動は最も保守的に**：迷ったら何もしない。書き換え／削除より「読み出し専用で別場所にコピー」

#### 多重防御策（実装必須）
- [ ] **マイグレーション/初回同期前の自動スナップショット**：SQLite ファイルそのものを `memolette.sqlite.bak.YYYYMMDD-HHmm` で別ファイルにコピー。失敗時は自動ロールバック
- [ ] **2-phase commit**：先にクラウドへアップロード → サーバー側で件数・ハッシュ検証 → 検証通った時点で「同期済み」マーク。ローカルデータの削除/上書きはしない
- [ ] **件数・ハッシュチェック**：アップロード前後で `SELECT COUNT(*)` と全レコードのハッシュ比較。1 件でもズレたらアボート＋ロールバック
- [ ] **ユーザーに見える定期エクスポート**：1 日 1 回、JSON/SQLite を Documents ディレクトリ（iOS Files から見える場所）に書き出し。アプリが死んでもユーザー自身で救出可能
- [ ] **アンインストール耐性**：iCloud Drive 等、アプリ削除しても残る場所にバックアップを配置
- [ ] **マイグレーション前の必須確認 UI**：「○件のメモをクラウドへ。バックアップは Documents/memolette-backup-XXXX.db に保存しました」と明示。ユーザー承認なしには進まない
- [ ] **アプリ内ロールバック UI**：「設定 > データ復元」で過去のバックアップから戻せる
- [ ] **テスト網羅**：移行ロジックはユニット＋統合テスト必須。テストなしでリリース不可

#### 同期機能本体
- [ ] Firebase連携（認証・Firestore）
- [ ] SQLite ↔ Firestore 同期ロジック（上記方針に厳格準拠）
- [ ] iCloud/CloudKit 同期
- [ ] オフライン対応
- [ ] コンフリクト解決（最終更新優先 + 競合履歴を保持）

## Phase 16: 全画面サイズ対応のレスポンシブ化（リリース前必須）

### スコープ
- [ ] **メモカード**（`memo_card.dart`）の `_titleFont` / `_bodyFont` / `_cardPadding` 等を「カード実幅から派生する比率計算」に変更（`LayoutBuilder` で実幅取得）
- [ ] **ToDoカード**（`todo_card.dart`）も同様に比率化（しおりサイズ、タイトルフォント、件数バッジ等）
- [ ] **GridSizeOption** 各値（grid3x6/2x5/2x3/1x2/1flex/titleOnly）を画面幅に応じて適切に表示／メニュー項目側でも対応
- [ ] **カレンダー月間ビュー**（`calendar_view.dart` 内 `_MonthBlock`）の日付セル高さ・フォントを画面幅依存に
- [ ] **DayItemsPanel** のヘッダ／カードフォント・パディング
- [ ] **メモ入力エリア**（`memo_input_area.dart`）の高さ計算が縦幅（特に SE 系の 667pt）でも収まるように
- [ ] **検証マトリクス**: 全 gridSize × 全画面幅クラス（320/375/390-393/430+）で見た目すり合わせ
