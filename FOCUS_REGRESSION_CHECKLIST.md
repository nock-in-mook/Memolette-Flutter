# フォーカス／編集モード／キーボード リグレッション・チェックリスト

**目的**: グローバル `NavigatorObserver.didPop` で一律 `unfocus()` を入れる場合に、
過去調整した「フォーカス・編集モード突入・キーボード・ダイアログ閉じ後の挙動」が
リグレッションしていないかを、リリース前に**実機で 1 つずつ確認**する。

**チェック方法**: 各項目を実機で再現 → `[x]` を付ける。問題があったらその項目の
脇に `← 問題: ○○` と書いて、後でまとめて対処。

**注意**: グローバル `unfocus on pop` を入れると、特に **A セクション**（編集中の
ダイアログ閉じ後の編集続行）と **C セクション**（画面遷移からの戻り）の挙動が
変わる可能性が高い。**「閉じた後にカーソル / キーボードが消える」が新挙動**になる。
それで困るのが許容範囲か、要件次第。

## A. 編集中ダイアログ閉じ後の挙動（`focusSafe` 不使用 + `_isDialogOpen` フラグ系）

> 出典: `HANDOFF.md` 「ダイアログ設計の定石（#24 で確立）」 / `SESSION_HISTORY.md:519`
>
> 過去調整: 編集中ダイアログは `focusSafe` を使わず、Navigator の自動フォーカス復元で
> キャンセル後に編集カーソルを戻す設計。グローバル `unfocus on pop` を入れると **戻らなくなる**。

- [ ] **A1. 本文クリア確認ダイアログ**: 本文編集中 → 本文クリアボタン → 確認ダイアログ → **キャンセル** → カーソルが本文の元位置に戻る／キーボード残ってる
- [ ] **A2. メモ削除（長押し or ゴミ箱）の確認ダイアログ**: 編集中 → 削除 → キャンセル → カーソル戻る
- [ ] **A3. 消しゴム（線で消す）の確認ダイアログ**: 編集中 → 消しゴム → キャンセル → 編集続行
- [ ] **A4. ToDo編集中のリセット確認ダイアログ**: ToDoアイテム編集中 → リセット → キャンセル → 編集続行
- [ ] **A5. ToDoリスト削除確認ダイアログ**: 編集中 → 削除 → キャンセル → 編集続行
- [ ] **A6. タグ削除確認ダイアログ**（メモ 0 件のとき確認だけ）: 確認後の挙動
- [ ] **A7. 色選択ダイアログ閉じ後**: 編集中に色選択ダイアログを出して色を選ぶ／キャンセル → 編集続行

## 確認方法メモ

- 各項目を **iPhone 実機（USB release）** で確認するのが基本
- 細かい挙動はシミュ（iPhone 17 Pro / Pro Max）でも追える
- iPad 横画面系（C10, C11, M セクション）は iPad 実機 or iPad Pro 13-inch シミュで
- 問題見つけたらこのファイルの該当行に `← 問題: ○○` と書く

## 検証ステップの推奨順序

1. **C2（ユーザー報告: ToDo 開いて閉じるとメモ入力編集状態）** ← 最初に治ってるか確認
2. **A1〜A8**（ダイアログキャンセル後の編集続行）が壊れていないか
3. **D1〜D2**（新規メモ作成時の即フォーカス）が壊れていないか
4. **C1, C3〜C9**（各画面遷移後）の挙動
5. 残り全項目

## カバー範囲の限界

- ここに載ってない箇所で問題見つけたら、その場でこのファイルに追記してください
