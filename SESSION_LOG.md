原因と修正案を整理する。

## バグの原因
1. ユーザーが項目X編集中（テキスト「abc」入力済み）
2. 「項目追加」ボタンタップ → `_createItem` が空タイトルでY を DB 挿入 + `setState`で `_editingItemId = Y` に切替
3. setStateで X 行は `isEditing=false` に → 表示は `displayTitle = optimistic ?? item.title` を計算
4. しかし**この瞬間 `_optimisticTitles[X.id]` も DB の `item.title` も空** → 「（空のアイテム）」表示
5. その後、X の `_EditingItemField` が unmount → dispose で `onCommit` → `_commitEditWithText` で DB 更新＆ optimistic セット → 「abc」表示に戻る

**画面遷移: 「abc」 → 「（空のアイテム）」 → 「abc」** が見える。

## 修正案
`_EditingItemField` の `TextField.onChanged` で常時 `_optimisticTitles[item.id] = text` を更新しておく。setStateは不要（編集中は楽観値は表示に使われない）。`_editingItemId` が切り替わった瞬間、すでに最新テキストが楽観値に入っているので「（空のアイテム）」表示を回避。

### 正常ケース確認（回帰防止）
- A. 編集中 → 「項目追加」 → 修正対象、abc保持されて表示
- B. 全消し → 「項目追加」 → optimistic='' → 削除 → アイテム消える（自然）
- C. キーボード「次へ」(onSubmitted) → 既存の chain 経路で commit、変化なし
- D. 別項目をタップ → 既存の dispose 経路で commit、変化なし
- E. メモボタン → `_toggleMemo` で `_editingItemId=null` → dispose 経路、変化なし
- F. onTapOutside → 既存通り何もしない、変化なし
