# ロードマップ — Memolette Flutter版

## 設計方針（Swift版から引き継ぎ・絶対的命題）
- **UIは常にシンプルを保つ** — 機能が増えてもカオスにしない
- 普通の人にもものすごく使いやすく、パワーユーザーの期待も大きく上回る
- 「鳥肌が立つほど便利！」と言わせるアプリにする
- タブ内のメモ一覧は「バッグ」と呼ぶ（開発用語として統一）

## Phase 8: iPad対応（★優先／同期テストの前提）
ユーザーが iPad しか所持していないため、Phase 9 同期検証の前に必須。
B（スプリットビュー）で実用価値を出してから C の要素を段階的に追加する方針。

### Step 0: 事前課題（着手前に潰す）
- [x] 横画面回転は実機で動作確認済（iPad 実機 iOS 26.2.1）。シミュレータは既知バグで回転追従しないため、開発用に Info.plist `UIRequiresFullScreen=true` を一時的に有効化してシミュで回転動作を取り戻している。**Phase 8 完了後に外すこと**（Split View / Slide Over / Stage Manager を復活）
- [x] 設計判断は確定済
  - 縦画面 iPad: iPhone 同等の単列レイアウト（Step A 実装済）
  - スプリットビュー: 横画面のみで固定比率（1:1 + `VerticalDivider`）
  - タブレット判定: `isTablet = shortestSide >= 600`、`isWide = width >= 840 && width > height`

### Step A: レスポンシブ基盤（B の土台）
- [ ] タブレット判定ユーティリティ（`isTablet`, `isWideLayout` helper）
- [ ] MediaQuery / LayoutBuilder でブレークポイント設計
- [ ] コンテンツの max-width 制限（入力欄・フッターツールバーを中央寄せ）
- [ ] メモグリッド列数の動的化（iPad幅で 3〜4 列）
- [ ] 絶対値→相対値化（ROADMAP Phase 14 から前倒し）

### Step B: スプリットビュー（横画面の主役）
- [ ] 横画面 or 幅広時は左右2カラム
  - 左: タブ＋バッグ一覧／右: メモ詳細
  - Swift版 MainView.swift を参考に設計
- [ ] 1メモ編集中も一覧が見えたまま編集できる動線
- [ ] Multitasking / Stage Manager の動的リサイズに追従（起動中にも画面幅が変わる。最大の盲点）
- [ ] iPadOS Split View / Slide Over での破綻チェック

### Step C: iPad らしさの付加価値（優先度順）
- [ ] **⌘キーショートカット** — `⌘N` 新規、`⌘F` 検索、`⌘Z`/`⇧⌘Z` Undo/Redo、`⌘B/I` 太字/斜体、`⌘1-9` タブ切替
- [ ] **ドラッグ&ドロップ** — 他アプリから画像/テキストを投げ込んで即メモ化、メモ→他アプリへのエクスポート
- [ ] **右クリック / 長押しコンテキストメニュー** — 編集・削除・タグ付け
- [ ] **サイドバー常時表示** — 親タグ一覧を左固定（NavigationRail 的な佇まい）
- [ ] **並列編集モード** — 画面を左右分割して別メモを並べる（アイデアメモから昇格）
- [ ] **Apple Pencil Scribble 対応** — ブロックエディタ内で TextField が正しく受け付けるか検証
- [ ] **マルチウィンドウ（iPadOS Scene）** — 同じアプリを2画面で開く（余力があれば）

### UI 微調整（実機スクショで見つけた破綻）
- [ ] 入力エリアが横長すぎて空白多い（max-width 制限）
- [ ] メモカードが iPhone と同じ2列で間延び（列数動的化）
- [ ] サブフィルタ（すべて/よく見る/最近/タグなし）が等間隔分散で横に広がりすぎ
- [ ] フッターツールバーが左寄り、右半分が空白
- [ ] 全体的な間延び感 — 余白と最大幅を制御（フォントサイズは固定でOK）
- [ ] ルーレットUIの配置 — iPad だと腕動作が大きくなる。片手の親指が届く位置に寄せる
- [ ] タブの無限スクロール — iPad幅だとタブが余る。見せ方を再設計

### 盲点 / 要検証
- [ ] フローティングキーボードで `viewInsets.bottom` が 0 でも隠れるケース
- [ ] BTキーボード接続時のショートカット・フォーカス挙動
- [ ] トラックパッド hover / secondary tap
- [ ] Safe Area / Home indicator が縦横切替で動的変化
- [ ] 画像サムネの密度（画面が広い分、小サムネが点のように見える恐れ）

### 完了条件
- [ ] iPadシミュレータ（Pro 12.9 / 11 / mini / Air）で破綻なく動く
- [ ] iPad 実機でビルド＆動作確認（ケーブル入手後）
- [ ] 縦横切替、Split View、Stage Manager でも崩れない
