## 原因

**Flutterのモーダル仕様** — `showDialog` や `showModalBottomSheet` を閉じる瞬間に、Navigator が「直前のフォーカス」を自動復元する。結果：

1. メモ長押し → `FocusScope.unfocus()` でキーボード閉じる
2. ユーザーがキャンセル → モーダル閉じる
3. Navigator が閉じるタイミングで直前のFocusNode（入力欄）を復元 → キーボード再出現

## 既存コードの状況

`home_screen.dart` のタブ長押し系（`_showSpecialTabActions` / `_showTagActions`）には**すでに対策済み**で、コメント付きのこのパターンを使ってる：

```dart
// ダイアログが直前のフォーカス（テキスト入力欄）を復元しないように
FocusScope.of(context).unfocus();
// 次フレームでも念押しでフォーカス解除（ダイアログ復元との競合防止）
WidgetsBinding.instance.addPostFrameCallback((_) {
  if (mounted) FocusScope.of(context).unfocus();
});
```

`_showMemoActions` にはこの「次フレーム unfocus」が**抜けている**。これが直接の原因。

しかも、アプリ全体で `showDialog`系が **30箇所近く**ある（settings, todo_*, quick_sort など）。それぞれで個別に対策してると絶対漏れる。

## 完全予防策の提案

**共通ラッパーを作る**（`lib/utils/safe_dialog.dart` みたいな場所に）：

```dart
Future<T?> showSafeDialog<T>(...) async {
  FocusScope.of(context).unfocus();      // 開く前
  final result = await showGeneralDialog<T>(...);
  if (!context.mounted) return result;
  FocusScope.of(context).unfocus();      // 閉じた直後
  WidgetsBinding.instance.addPostFrameCallback((_) {
    if (context.mounted) FocusScope.of(context).unfocus();  // 次フレーム念押し
  });
  return result;
}
```

同様に `showSafeBottomSheet`、`showSafeCupertinoModal` も作る。既存の呼び出しを全部これに置き換えれば、今後も含めて完全予防。
